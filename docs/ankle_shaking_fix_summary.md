## 如果脚踝仍然抖动，可以尝试：

### 方法1: 进一步降低脚踝PD增益

```python
# 在 test_mpc_rl.py 中修改
kp_ankle = 30.0    # 进一步降低位置增益
kd_ankle = 40.0    # 进一步增加阻尼
```

### 方法2: 检查重力补偿数值

在首帧调试输出中检查脚踝关节的重力补偿力矩是否合理：
- 正常情况下，脚踝的重力补偿力矩应该较小（通常 < 10 N·m）
- 如果重力补偿力矩过大，可能需要检查模型或初始姿态

### 方法3: 调整初始姿态

脚踝的初始姿态可能不稳定，可以尝试：
```python
# 在 INITIAL_POSE 中调整脚踝角度
INITIAL_POSE = np.array([
    ...
    -0.05, 0.0,  # left_ankle_pitch, left_ankle_roll
    ...
    -0.05, 0.0,  # right_ankle_pitch, right_ankle_roll
    ...
])
```

### 方法4: 添加低通滤波

对脚踝关节的控制力矩添加低通滤波，减少高频抖动：
```python
# 添加低通滤波器
alpha = 0.7  # 滤波系数（0-1，越大越平滑）
tau_ankle_filtered = alpha * tau_ankle_prev + (1 - alpha) * tau_ankle
```

### 方法5: 检查控制频率

如果控制频率过低，可能导致控制延迟和抖动：
- 当前 MuJoCo timestep 通常是 0.002s（500Hz）
- 确保控制循环能够跟上这个频率

## 调试建议

### 1. 观察调试输出

运行脚本时，首帧会输出脚踝关节的控制信息：
```
[调试] left_ankle_pitch_joint: 重力补偿=X.XXXX, PD力矩=X.XXXX, 总力矩=X.XXXX
```

检查：
- 重力补偿力矩是否合理
- PD力矩是否过大
- 总力矩是否在合理范围内

### 2. 监控脚踝关节状态

可以添加实时监控：
```python
if frame_count % 100 == 0:  # 每100帧打印一次
```

### 3. 检查是否还有其他问题

- **机器人整体不稳定**：可能是整体控制策略问题，不仅仅是脚踝
- **特定方向抖动**：可能是某个轴的PD参数需要单独调整
- **周期性抖动**：可能是控制频率或数值积分问题
