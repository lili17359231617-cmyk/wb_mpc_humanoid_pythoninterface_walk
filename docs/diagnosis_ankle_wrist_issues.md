# 机器人脚踝抖动和手腕乱摆问题诊断报告

## 问题描述

1. **脚踝抖动明显**
2. **手腕无力乱摆**
3. **机器人会倒下**

## 问题分析

### ✅ 已确认：映射和通路正确

从终端输出和代码分析可以确认：
- MPC接口初始化成功
- 状态映射正确（MPC状态维度58，关节数23）
- 控制通路流畅（MPC求解成功，控制信号正常输出）

### ❌ 发现的关键问题

#### 1. **手腕关节未应用控制（已修复）**

**问题位置**：`test_mpc_rl.py` 第242-248行

**问题描述**：
- 虽然计算了手腕关节的PD力矩（`tau_pd[i]` for i in range(23, 29)）
- 但是**没有将控制信号应用到MuJoCo的actuator上**
- 代码只遍历了`mpc_ctrl_idxs`（23个MPC关节），忽略了手腕关节的6个actuator

**影响**：
- 手腕关节完全不受控制，会自由摆动
- 手腕的摆动会影响整体平衡，可能导致机器人倒下

**修复方案**：
```python
# 添加手腕关节控制应用
for i in range(MPC_JOINT_DIM, ALL_JOINT_DIM):
    act_id = all_ctrl_idxs[i]
    torque = tau_pd[i]  # 手腕关节只使用PD控制
    max_torque = 20.0  # 手腕关节力矩限制更小
    torque = np.clip(torque, -max_torque, max_torque)
    data.ctrl[act_id] = torque
```

#### 2. **脚踝抖动的原因（已修复）**

**✅ 已确认：重力补偿索引错误**

**问题描述**：
- 验证发现：22个关节的重力补偿索引映射错误
- 当前代码：`tau_gravity = data.qfrc_inverse[6:].copy()`
- 问题：`qfrc_inverse[6:]`的索引顺序与`mpc_qpos_idxs`的顺序不一致
- 结果：重力补偿力矩应用到错误的关节上，导致脚踝等关节抖动

**修复方案**（已实施）：
```python
# 修复前（错误）：
tau_gravity = data.qfrc_inverse[6:].copy()

# 修复后（正确）：
tau_gravity = np.array([data.qfrc_inverse[mpc_qpos_idxs[i]]
                        for i in range(len(mpc_qpos_idxs))])
```

**其他可能原因**：
- **PD增益不合适**：如果修复后仍有抖动，可以尝试调整PD参数
  - 当前设置：`kp_mpc = 200.0`, `kd_mpc = 15.0`
  - 脚踝关节可能需要更低的kp和更高的kd来减少振荡
- **控制频率问题**：如果控制频率过低，可能导致控制延迟

#### 3. **机器人倒下的原因**

**主要原因**：
1. **手腕未控制**：手腕乱摆会影响整体平衡（已修复）
2. **脚踝抖动**：脚踝不稳定会导致支撑不稳定
3. **整体控制策略**：当前只使用重力补偿+PD控制，没有MPC的主动平衡控制

**注意**：
- 当前代码中MPC只在第0帧运行一次，之后没有持续更新
- 这意味着机器人没有MPC的主动平衡能力，只能依靠PD控制维持姿态
- 如果初始姿态不稳定或受到扰动，机器人容易倒下
