# 首次打开容器 / 迁移到新设备

本文档说明在**新机器上首次使用**或**从其他设备迁移**本项目时，如何安装依赖并完成首次编译，便于在不同设备上复现环境。

---

## 一、环境要求

| 项目 | 要求 |
|------|------|
| 操作系统 | Ubuntu 22.04（推荐，与 ROS2 Jazzy 一致） |
| 内存 | 建议 ≥16GB；首次编译 16GB 用 2 并行，32GB 用 4，64GB 用 6 |
| 磁盘 | 至少 10GB 可用（源码 + 编译产物 + 依赖） |
| Docker（容器方式） | Docker Engine + 可选：VS Code/Cursor + Dev Containers 扩展 |
| 网络 | 能访问 apt / pip 源（拉取依赖） |

---

## 二、依赖清单（便于在其他设备复现）

### 2.1 系统依赖（apt）

由 `dependencies.txt` 定义，需在 **ROS2 已安装** 的前提下使用（`ROS_DISTRO` 会替换为 `jazzy` 等）：

- 构建/格式：`clang-format`, `make`, `build-essential`, `ninja-build`, `ccache`, `black`
- 数学/图形：`libeigen3-dev`, `libglpk-dev`, `libboost-*`, `libglfw3-dev`, `liburdfdom-dev`, `libabsl-dev`, `libxinerama-dev`, `libxcursor-dev`, `libxi-dev`
- Python 相关：`python3-pygame`, `python3-tk`
- ROS2 包：`ros-${ROS_DISTRO}-ament-cmake-clang-format`, `ros-${ROS_DISTRO}-joint-state-publisher-gui`, `ros-${ROS_DISTRO}-xacro`, `ros-${ROS_DISTRO}-pinocchio`, `ros-${ROS_DISTRO}-rviz2` 等（见 `dependencies.txt`）

**安装方式（仅本地安装时使用）：**

```bash
# 先安装 ROS2 Jazzy，再在项目根目录执行
cd /path/to/wb_humanoid_mpc
export ROS_DISTRO=jazzy
envsubst < dependencies.txt | xargs sudo apt install -y
```

### 2.2 Python 依赖（pip）

**根目录** `requirements.txt`（一键安装，已覆盖以下全部）：

| 包 | 用途 |
|----|------|
| numpy, scipy | 数值计算、姿态变换（全项目） |
| gymnasium | RL 环境 G1MpcEnv / G1MpcWeightEnv |
| mujoco | Python MuJoCo 仿真（G1MpcEnv、sim_interface、test_mpc_PD+G.py） |
| stable_baselines3 | PPO 训练（train_g1_ppo_v1.py），会拉取 torch |

**子包**（脚本 `install_python_deps.sh` 会一并安装，或按需单独安装）：

- `humanoid_nmpc/humanoid_common_mpc_pyutils/requirements.txt`：pandas, vpselector
- `humanoid_nmpc/remote_control/requirements.txt`：pygame, customtkinter（遥控 GUI）

**子模块**（仅在使用时安装）：`lib/ocs2_ros2/ocs2_mpcnet/ocs2_mpcnet_core/requirements.txt` 含 torch、tensorboard 等，用于 MPCNet 训练。

**一键安装所有 Python 依赖（在项目根目录）：**

```bash
cd /path/to/wb_humanoid_mpc
# 使用脚本（推荐）
chmod +x .devcontainer/install_python_deps.sh
./.devcontainer/install_python_deps.sh
# 若需要遥控 GUI：INSTALL_REMOTE_CONTROL=1 ./.devcontainer/install_python_deps.sh
# 容器内系统 Python：PIP_EXTRA="--break-system-packages" ./.devcontainer/install_python_deps.sh
```

或手动安装：

```bash
pip3 install --user -r requirements.txt
pip3 install --user -r humanoid_nmpc/humanoid_common_mpc_pyutils/requirements.txt
# 若需要遥控 GUI：pip3 install --user -r humanoid_nmpc/remote_control/requirements.txt
```

容器内若使用系统 Python 且需写系统 site-packages，可用 `pip3 install --break-system-packages -r requirements.txt`（与 Dockerfile 一致）。

## 三、三种使用方式

### 方式 A：Dev Container（VS Code / Cursor，推荐）

1. 安装 [Dev Containers](https://marketplace.visualstudio.com/items?itemName=ms-vscode-remote.remote-containers) 扩展。
2. 用 VS Code/Cursor 打开**工作空间根目录**（即包含 `wb_humanoid_mpc` 的 colcon 工作空间，或本仓库根目录）。
3. 命令面板（`Ctrl+Shift+P`）→ **“Dev Containers: Rebuild and Reopen in Container”**。
4. 等待镜像构建完成（会按 `docker/Dockerfile` 安装系统依赖与根目录 `requirements.txt`）。
5. 容器打开后，在终端执行**首次编译**（见第四节）。

**注意**：`devcontainer.json` 中的 `USER_ID`/`GROUP_ID` 等会从本机传入，便于挂载卷权限一致；若迁移到新设备，在新设备上直接 “Rebuild and Reopen in Container” 即可。

### 方式 B：仅用 Docker 脚本（不依赖 VS Code）

1. **构建镜像**（与 devcontainer 使用同一 Dockerfile）：

   ```bash
   cd /path/to/wb_humanoid_mpc/docker
   ./image_build.bash
   ```

2. **启动容器**：

   ```bash
   ./launch_wb_mpc.bash
   ```

3. 进入容器后，从**第四节**开始做子模块初始化与首次编译。

### 方式 C：本地安装（不用 Docker）

1. 安装 ROS2 Jazzy：[官方文档](https://docs.ros.org/en/jazzy/Installation/Ubuntu-Install-Debs.html)。
2. 安装系统依赖（见 2.1）。
3. 安装 Python 依赖（见 2.2）。
4. 克隆仓库并初始化子模块，然后按第四节编译：

   ```bash
   git clone --recursive <repo_url> wb_humanoid_mpc
   cd wb_humanoid_mpc
   git submodule update --init --recursive
   ```

## 四、首次编译（容器内或本机通用）

在**项目根目录**（`wb_humanoid_mpc`，即包含 `Makefile` 的目录）执行：

```bash
# 1. 设置编译环境（加载 ROS、子模块、环境变量）
source .devcontainer/setup_build_env.sh

# 2. 按内存设置并行数（必选）
export PARALLEL_JOBS=2   # 16GB=2, 32GB=4, 64GB=6

# 3. 全量编译
make build-all

# 4. 加载安装目录（每次新开终端建议执行）
source /wb_humanoid_mpc_ws/install/setup.bash
# 若为本地安装，改为：
# source /path/to/your_ws/install/setup.bash
```

**可选**：把 `source .../install/setup.bash` 和（若需要）`export PARALLEL_JOBS=2` 写入 `~/.bashrc`，避免每次手动执行。

## 六、如何检验迁移是否成功

在**新设备上**完成克隆、依赖安装与首次编译后，运行**迁移验证脚本**即可一次性检查环境与编译结果是否就绪。

### 一键验证（推荐）

在项目根目录（`wb_humanoid_mpc`）执行：

```bash
cd /path/to/wb_humanoid_mpc
chmod +x .devcontainer/verify_migration.sh
./.devcontainer/verify_migration.sh
```

若使用容器且工作空间不在默认路径，可指定安装目录：

```bash
INSTALL_DIR=/wb_humanoid_mpc_ws/install ./.devcontainer/verify_migration.sh
```

**脚本会检查：**

| 检查项 | 说明 |
|--------|------|
| ROS2 | 是否已安装 jazzy/humble |
| 编译工具 | colcon、cmake、python3、make 是否可用 |
| Python 依赖 | numpy、scipy、gymnasium、mujoco、stable_baselines3 是否可导入 |
| Git 子模块 | 是否已 `git submodule update --init --recursive` |
| 安装目录 | `install/setup.bash` 及 hpipm/blasfeo 库是否存在（编译后） |
| humanoid_wb_mpc_py | source install 后能否 `import humanoid_wb_mpc_py` |

**结果含义：**

- 脚本**退出码 0** 且输出「全部通过」→ 可认为迁移成功，能正常编译与运行 MPC/RL 相关脚本。
- 若有 **[失败]** 或 **[跳过]**：按脚本提示补装依赖、初始化子模块或完成编译后再次运行。

### 手动逐项验证

若不想用脚本，可按 [五、验证](#五验证) 与下面命令自行检查：

```bash
# 环境与工具
echo $ROS_DISTRO
which colcon cmake python3 make

# Python 依赖
python3 -c "import numpy, scipy, gymnasium, mujoco, stable_baselines3; print('deps OK')"

# 编译后（需先 source install/setup.bash）
python3 -c "import humanoid_wb_mpc_py; print('mpc_py OK')"
ls /wb_humanoid_mpc_ws/install/hpipm_catkin/lib/
```

## 七、迁移到新设备检查清单

本项目**依赖已全部写入** `dependencies.txt`（系统）与 `requirements.txt`（Python），按本文安装即可在新设备复现环境。

- [ ] 在新设备上安装 Docker（或 ROS2 + 依赖）
- [ ] 克隆仓库并执行 `git submodule update --init --recursive`
- [ ] 使用方式 A/B 构建并进入容器，或方式 C 安装系统与 Python 依赖
- [ ] 执行 `source .devcontainer/setup_build_env.sh` 并设置 `PARALLEL_JOBS`
- [ ] 执行 `make build-all` 并 `source install/setup.bash`
- [ ] 运行 **`.devcontainer/verify_migration.sh`** 检验迁移是否成功

更多编译问题可参考：`.devcontainer/BUILD_INSTRUCTIONS.md`、`.devcontainer/QUICK_START.md`、`.devcontainer/FIX_PERMISSIONS.md`。
